Lecture 3: Ethical Web Principles & Accessibility

Web Ethics Fundamentals

Core principles for ethical web development:

  • Privacy: Protect user data (GDPR, COPPA compliance)
  • Transparency: Disclose data collection practices
  • Inclusivity: Design for diverse audiences
  • Sustainability: Optimize for energy efficiency
  • Honesty: Avoid dark patterns (misleading UI)

Web Accessibility (WCAG)

WCAG 2.1 Guidelines (POUR principles):

  1. Perceivable:
    • Text alternatives for non-text content
    • Captions for multimedia
    • Contrast ratio ≥ 4.5:1 for normal text
  2. Operable:
    • Keyboard navigable interfaces
    • No content that causes seizures (≤3 flashes/sec)
  3. Understandable:
    • Clear error messages in forms
    • Consistent navigation
  4. Robust:
    • Compatible with assistive technologies
    • Valid HTML/CSS

Implementing Accessibility

Key HTML5 techniques:

<!-- Semantic HTML -->
<nav aria-label="Main navigation">...</nav>

<!-- ARIA attributes -->
<button aria-expanded="false">Menu</button>

<!-- Alt text best practice -->
<img src="chart.jpg" alt="Sales growth chart: 15% increase Q1 2024">

Ethical Dilemmas in Web Dev

Common challenges:

  • Cookie Consent: Balancing usability vs. compliance
  • Algorithmic Bias: Ensuring AI doesn't discriminate
  • Environmental Impact: Reducing carbon footprint of websites
  • Open Source: Proper attribution of code
